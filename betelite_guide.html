üéØ Platform Overview

BetElite is a comprehensive eSports betting platform built with PHP and MySQL, featuring real-time odds, secure payment integration with Paystack, and a modern responsive design.

Key Features

  • Real-time eSports match betting
  • Live odds calculation and updates
  • Secure user authentication and authorization
  • Paystack payment gateway integration
  • Wallet system with transaction history
  • Admin dashboard for match and user management
  • Responsive mobile-first design
  • RESTful API architecture

üöÄ Installation Steps

Step 1: Install XAMPP

Download and install XAMPP from the official website. Choose the version that matches your operating system.

Windows: Run xampp-windows-x64-installer.exe
Mac: Run xampp-osx-installer.dmg
Linux: sudo chmod +x xampp-linux-x64-installer.run && sudo ./xampp-linux-x64-installer.run

Step 2: Start Apache and MySQL

Open XAMPP Control Panel and start both Apache and MySQL services.

‚úì Apache should be running on port 80
‚úì MySQL should be running on port 3306

Step 3: Extract BetElite Files

Extract the BetElite platform files to the XAMPP htdocs directory:

Windows: C:\xampp\htdocs\betelite\
Mac/Linux: /Applications/XAMPP/htdocs/betelite/ or /opt/lampp/htdocs/betelite/

Step 4: Set Permissions

For Mac/Linux, set proper permissions:

sudo chmod -R 755 /path/to/betelite/
sudo chmod -R 777 /path/to/betelite/uploads/

üóÑÔ∏è Database Setup

Creating the Database

Open phpMyAdmin (http://localhost/phpmyadmin) and create a new database:

CREATE DATABASE betelite_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

Import Database Schema

Import the provided SQL file to set up all tables:

  1. In phpMyAdmin, select the betelite_db database
  2. Click on the "Import" tab
  3. Choose the file: betelite_db.sql
  4. Click "Go" to import

Database Structure

Table Name Purpose Key Fields
users User accounts id, username, email, password, wallet_balance
matches eSports matches id, team1, team2, game_type, match_time, status
bets User bets id, user_id, match_id, bet_amount, bet_team, odds
transactions Payment history id, user_id, type, amount, status, reference
odds Betting odds id, match_id, team1_odds, team2_odds, updated_at
‚ö†Ô∏è Important: After importing, verify that all tables are created successfully. Check that there are no errors in the import log.

‚öôÔ∏è Configuration

Database Configuration

Edit the config.php file in the root directory:

<?php // Database Configuration define('DB_HOST', 'localhost'); define('DB_USER', 'root'); define('DB_PASS', ''); define('DB_NAME', 'betelite_db'); // Create connection $conn = mysqli_connect(DB_HOST, DB_USER, DB_PASS, DB_NAME); // Check connection if (!$conn) { die("Connection failed: " . mysqli_connect_error()); } // Set charset to utf8mb4 mysqli_set_charset($conn, "utf8mb4"); ?>

Site Settings

Configure your site settings in settings.php:

<?php // Site Configuration define('SITE_NAME', 'BetElite'); define('SITE_URL', 'http://localhost/betelite'); define('ADMIN_EMAIL', 'admin@betelite.com'); // Timezone date_default_timezone_set('Africa/Accra'); // Session Configuration ini_set('session.cookie_httponly', 1); ini_set('session.use_only_cookies', 1); session_start(); ?>

Environment Variables

Create a .env file for sensitive data (recommended for production):

# Database DB_HOST=localhost DB_USER=your_username DB_PASS=your_password DB_NAME=betelite_db # Paystack PAYSTACK_SECRET_KEY=sk_test_xxxxxxxxxxxxx PAYSTACK_PUBLIC_KEY=pk_test_xxxxxxxxxxxxx # Site SITE_URL=https://yourdomain.com DEBUG_MODE=false
üîí Security Note: Never commit your .env file or config.php with real credentials to version control. Add them to .gitignore immediately.

üîå API Endpoints

Authentication API

Endpoint Method Description
/api/auth.php?action=register POST Register new user
/api/auth.php?action=login POST User login
/api/auth.php?action=logout POST User logout
/api/auth.php?action=verify GET Verify user session

Betting API

Endpoint Method Description
/api/bets.php?action=place POST Place a new bet
/api/bets.php?action=list GET Get user's bets
/api/bets.php?action=cancel POST Cancel pending bet

Example API Request

Placing a bet via JavaScript:

// Place Bet Example fetch('/api/bets.php?action=place', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ match_id: 123, bet_amount: 50.00, bet_team: 'team1', odds: 1.85 }) }) .then(response => response.json()) .then(data => { if(data.success) { alert('Bet placed successfully!'); } else { alert('Error: ' + data.message); } }) .catch(error => console.error('Error:', error));

üí≥ Paystack Integration

Setup Paystack Account

  1. Visit paystack.com and create an account
  2. Verify your business information
  3. Navigate to Settings ‚Üí API Keys & Webhooks
  4. Copy your Test and Live API keys

Configure Paystack Keys

Add your Paystack keys to the configuration:

<?php // Paystack Configuration define('PAYSTACK_SECRET_KEY', 'sk_test_your_secret_key_here'); define('PAYSTACK_PUBLIC_KEY', 'pk_test_your_public_key_here'); // For production, use live keys: // define('PAYSTACK_SECRET_KEY', 'sk_live_your_secret_key_here'); // define('PAYSTACK_PUBLIC_KEY', 'pk_live_your_public_key_here'); ?>

Deposit Implementation

// Initialize Paystack Payment function initializePayment(amount, email) { fetch('/api/payments.php?action=initialize', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ amount: amount * 100, // Convert to kobo email: email }) }) .then(response => response.json()) .then(data => { if(data.status) { // Redirect to Paystack payment page window.location.href = data.data.authorization_url; } }); }

Webhook Configuration

Set up webhook to receive payment notifications:

  1. In Paystack Dashboard, go to Settings ‚Üí Webhooks
  2. Add your webhook URL: https://yourdomain.com/api/webhook.php
  3. Select events: charge.success, transfer.success
  4. Save and copy the webhook secret
